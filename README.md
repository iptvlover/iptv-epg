# EPG 辅助调测工具

十年IPTV-EPG开发经验沉淀，专为一线开发、测试人员打造，基于Weinre二次开发，聚焦EPG调测核心痛点，提升调测效率，适配机顶盒专属场景。

## 📌 核心痛点与解决方案

### 问题一：接口调用受限，依赖IPTV认证

**低效率**：EPG接口需认证，且仅支持IPTV专用网络，跨环境调试不便。

**解决方案**：内置统一IPTV账号模拟登录，代理所有请求（静态资源+接口），脱离IPTV网络也可正常调试。

### 问题二：代码部署生效慢，效率低下

**低效率**：机顶盒端代码需部署至下发平台，生效需10分钟左右，严重拖慢开发节奏。

**解决方案**：基于Weinre二次开发的代理映射，让机顶盒直接访问本地代码，修改实时生效，告别等待。

### 问题三：接口调试繁琐，数据协调成本高

**低效率**：多业务系统接口调试需协调数据，查看接口需配置交换机抓包，操作复杂。

**解决方案**：基于Weinre二次开发实现接口篡改，拦截所有机顶盒请求，无需抓包，可自主修改测试数据。

## 🔧 基于Weinre二次开发核心优势

- 多环境多账号快速切换：支持专属环境配置，点击即可切换，避免冲突，适配多需求并行。

- 远程抓包与分享：实时抓取机顶盒HTTP/HTTPS请求，数据可共享，快速定位问题（如果是 HTTPS 需配置证书）。

- 轻量化易部署：Node环境即可通过npm安装启动，新员工快速上手，支持中心化管理。

- 灵活可扩展：支持多种配置规则及插件扩展，适配各类EPG调测场景，支持接口Mock。

- 高资源利用率：一台服务器可支持多团队、多用户独立使用，降低部署维护成本。

- console日志可视化：PC端可查看机顶盒console日志，直观定位前端问题；受硬件/内核影响，部分盒子存在兼容性限制，可通过兜底日志辅助排查。

## ✨ 工具核心特性

- IPTV统一账号代理，脱离专用网络可调试

- 本地代码实时生效，无需平台部署

- 无侵入接口篡改，无需交换机抓包

- 多环境切换、远程抓包、日志可视化

- 轻量化易上手，插件化可扩展，多端适配

## 📌 注意事项

- 抓取HTTPS请求需配置证书，确保客户端信任。

- 多人协作建议分配独立环境，避免规则冲突。

- 机顶盒与工具服务器需在同一局域网，避免连接异常。

- 接口篡改仅用于开发测试，禁止用于生产环境。

- 无需额外部署Weinre原生服务，安装工具即可使用。

- console日志存在兼容性限制，属行业共性问题，可通过兜底日志排查。
